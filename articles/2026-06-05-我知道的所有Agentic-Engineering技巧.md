## 摘要

本文整理了连续创业者 Matt Van Horn 总结的 22 条 Agentic Engineering 实战技巧，涵盖从规划、上下文管理、多 agent 调度、远程控制到开源贡献的完整工作流。核心理念是"先规划再动手"——用 /ce-plan 将模糊想法外化为计划文件，用 /ce-work 驱动执行，将 research → plan → build → review 变成默认流水线。文章强调拉开差距的关键在于上下文：截图、issue、会议录音、历史笔记持续喂入 agent 才能产生质变，人的角色从编码者上移至调度者与品味判断者。

## 关键要点
- 有想法先用 /ce-plan 生成 plan.md，计划是 agent 的"缰绳"
- 同时跑 4-6 个 agent 会话实现多线程调度，语音做主输入
- 上下文决定 agent 强弱：原始 transcript、last30days 研究、十年笔记全部喂入
- 任何做超过两次的事写成 skill，开源贡献纳入同一条 plan→build loop
- Agent 容易上瘾，越能 build 越要警惕，保持与真实世界的连接
